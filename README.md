# 房租收入管理系統原型

這個資料夾把原本的 `房租收入表.xlsx` 轉成可線上維護的前端系統原型。

## 使用方式

直接開啟 `index.html` 即可使用：

```text
C:\gomask_openai\rental-system\index.html
```

目前資料會保存在瀏覽器的 `localStorage` 作為本機快取，登入後可從 Supabase 載入並同步核心資料。公開部署時，真實租客資料應只存在 Supabase，不要寫進前端程式碼。

## 目前功能

- 年度與月份切換
- 本月應收、已收筆數、待追蹤、全年已收估算
- 已收、未收、逾期、特殊狀態顯示
- 收款日期與特殊狀態維護
- 物件、租客、租金、繳費週期、帳號與備註編輯
- 物件資料可記憶上次/本次電錶與水錶讀數，帳單報表可自動帶入
- 側邊欄可收合，讓收款表有更多橫向空間
- 匯款帳戶後五碼可輸入多組，並在收款表中完整顯示
- 新增與刪除物件
- 搜尋地址、租客與帳號
- 匯出年度 CSV
- 收款維護表可列印，列印時使用 A4 橫式並保留全年月份欄位
- 重置為 Excel 匯入的初始資料
- 退租結算模組：租金日割、押金抵扣、預繳租金退還、修繕/其他扣款
- 水電結算模組：官方兩月帳單總額、分錶用量比例、多租客/空屋分攤
- 租客帳單報表：依分錶比例產生可列印的租金與水電明細通知
- 多人協作：擁有者可邀請可編輯或只讀成員共同維護同一份資料
- 歷史水電資料：已匯入既有 Excel 帳期並統一轉為西元年，可套用官方電費/水費總額與分錶讀數
- 歷史帳期租客姓名以該月資料列最後一位姓名為準，避免把退租/換租備註合併成多人

## 退租結算邏輯

- 月租日割：月租金 ÷ 計租月份天數
- 退租日前租金：日租金 × 實住天數，退租日含當日
- 預繳租金應退：本月已收租金 - 退租日前租金
- 押金結算：押金 + 預繳租金應退 - 未繳租金 - 修繕/清潔扣款 - 其他扣款 - 退租租客水電分攤
- 水電帳單未收到前：水電列為「待結算」，不先進押金最終結算
- 水電帳單收到後：官方電費/水費總額 × 該租客分錶用量 ÷ 全部用量
- 空屋或屋主吸收期間：新增一列並選「空屋/屋主吸收」，該列會參與比例分攤但不向租客收費
- 多租客水電：第一列視為退租租客，用於押金結算；其他列用來計算同地址共用費分攤

## 租客帳單報表

- 帳單分為兩類：
- 簡易租金帳單：租金內含水電，或水電由租客自行向官方繳納，不另計水電
- 分錶水電帳單：租金不含水電，由房東繳官方帳單後，依分錶用量比例向租客收取
- 可從「歷史水電帳期」下拉套用既有 Excel 匯入資料
- 報表使用官方電費/水費總額作為母數
- 各租客電費：官方電費總額 × 該租客用電度數 ÷ 全部用電度數
- 各租客水費：官方水費總額 × 該租客用水度數 ÷ 全部用水度數
- 可輸入租金、其他費用、繳款期限與匯款資訊
- 地址可從物件資料下拉選取，並自動帶入租客與租金
- 新增帳單租客可從物件資料帶入租客、租金與分錶讀數
- 分錶水電報表會列出各分租單位本次/上次水電讀數、用量、分攤費用與應繳總額
- 分錶水電列印報表採用接近原 Excel 的三區塊布局：電費、水費、應繳房租
- 此戶總電費與此戶總水費會分別對齊在電費、水費區塊上方，分租單位列使用底色區分
- 帳期年份與月份使用下拉選單
- 繳款期限依物件繳費週期自動加 14 天預填
- 匯款銀行、帳號、收款人為關聯下拉資料，選任一欄會同步其他欄
- 列印時只會輸出正式報表，不會印出系統操作介面

## 下一階段建議

- GitHub Pages：依 `DEPLOY_GITHUB_PAGES.md` 部署
- Supabase：先依 `SUPABASE_SETUP.md` 執行 `supabase-schema.sql`
- 多人協作：既有專案需再執行 `supabase-migration-002-organizations.sql`
- 登入：先做單一房東管理帳號
- 資料同步：優先改造物件資料與每月收款紀錄
- 登入權限：管理者、只讀檢視者
- 通知：逾期自動提醒、LINE Notify 或 Email
- Excel 匯入：保留原檔上傳轉入資料庫
